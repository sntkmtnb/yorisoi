## 1. セグメント定義と推定人口構成

40代以上の婚活・パートナー探し市場のユーザー構成を推定し、10ペルソナをクラスタリング。

### セグメント一覧

| ID | セグメント名 | 代表ペルソナ | 推定構成比 | 推定人数 |
|----|------------|------------|----------|---------|
| S1 | IT慣れ層（経験者・批判的） | 佐藤健一, 松本あゆみ | 20% | 2,000人 |
| S2 | 安心感最重視層 | 田中美咲, 鈴木恵子, 小林千恵 | 30% | 3,000人 |

| S4 | 自信欠如・慎重層 | 高橋誠 | 10% | 1,000人 |

### セグメント別の詳細特性

#### S1: IT慣れ層（2,000人 / 20%）
- **属性:** ITリテラシー高い、他アプリ経験あり、批判的思考
- **代表:** 佐藤健一（47歳・IT管理職・Pairs挫折）、松本あゆみ（42歳・IT系・AI好き）
- **特徴:** 「どう違うのか」を論理的に理解したい。曖昧な表現を嫌う。比較データを求める。
- **LP離脱リスク:** 中（差別化が不明確だと離脱）
- **コンバージョン条件:** 具体的な仕組み説明、既存アプリとの明確な差別化

#### S2: 安心感最重視層（3,000人 / 30%）
- **属性:** アプリ未経験or嫌な体験あり、プライバシー懸念、慎重
- **代表:** 田中美咲（44歳・看護師）、鈴木恵子（48歳・離婚1年）、小林千恵（46歳・教師）
- **特徴:** 「出会い系＝危険」の先入観が強い。安全性の説明が不十分だと即離脱。利用規約を読む。
- **LP離脱リスク:** 高（安全性の訴求が弱いと8割離脱）
- **コンバージョン条件:** 匿名性保証、データ暗号化の明示、利用規約の透明性、他ユーザーの声

#### S3: IT苦手・シンプル重視層（3,000人 / 30%）
- **属性:** スマホ操作不慣れ、文字入力遅い、老眼、シンプルさ最優先
- **代表:** 山田太郎（52歳・経営者）、渡辺裕子（51歳・パート）、中村浩二（55歳・定年間近）
- **特徴:** 文字が小さいだけで離脱。複雑なUIは恐怖。入力フォームが最大のハードル。
- **LP離脱リスク:** 高（UI/フォントが不適切だと7割離脱）
- **コンバージョン条件:** 大きなフォント、最小限の入力、クイックリプライ、明確なCTA

#### S4: 自信欠如・慎重層（1,000人 / 10%）
- **属性:** 独身を恥じている、写真がない、行動に踏み出せない
- **代表:** 高橋誠（45歳・公務員・恥ずかしくて言えない）
- **特徴:** サービスの存在自体は知っていても登録に至らない。「自分には合わない」と思いがち。
- **LP離脱リスク:** 非常に高（心理的ハードルが最大の壁）
- **コンバージョン条件:** 写真不要の明示、匿名性、励ましコピー、小さな一歩感

#### S5: デザイン・体験重視層（1,000人 / 10%）
- **属性:** UI/UXに敏感、審美眼が高い、「ダサい」は即離脱
- **代表:** 伊藤大輔（43歳・フリーランスデザイナー）
- **特徴:** アニメーション、タイポグラフィ、カラーパレットの質で判断。
- **LP離脱リスク:** 中（デザイン品質で判断）
- **コンバージョン条件:** 洗練されたUI、スムーズなアニメーション、ブランドの一貫性

## 4. セグメント横断の重要発見

### 発見1: 「安心感」は全セグメントの最大公約数

1万人中、**推定7,000人（70%）が何らかの安心感の課題を持つ。**

| 安心感の種類 | 影響人数 | 影響率 | 対応状況 |
|------------|---------|--------|---------|
| 個人情報漏洩の不安 | 4,000人 | 40% | ✅ 対応済み（Trust Badge, 安心セクション, FAQ） |
| 「出会い系」への偏見 | 4,000人 | 40% | ✅ 対応済み（表現変更, 比較表） |
| 写真を出すことへの恐怖 | 3,000人 | 30% | ✅ 対応済み（写真不要明示, 公開タイミング説明） |
| 退会できるか不安 | 3,000人 | 30% | ✅ 対応済み（FAQ「退会はいつでも」） |
| 運営元の信頼性 | 2,500人 | 25% | ⚠️ 未対応（運営情報不足） |
| 課金トラップの不安 | 2,000人 | 20% | ✅ 対応済み（FAQ「料金」） |

### 発見2: 「入力負担」はS3+S4の4,000人に影響

| 入力の障壁 | 影響人数 | 対応状況 |
|-----------|---------|---------|
| テキスト入力自体が辛い | 3,000人 | ✅ クイックリプライ, 音声入力ヒント |
| 47都道府県のスクロール | 3,000人 | ⚠️ 未対応（地域ブロック分けが必要） |
| パスワード設定が面倒 | 3,000人 | △ 部分対応（強度表示あるが、ソーシャルログイン未対応） |
| selectボックス操作が難しい | 1,500人 | △ 部分対応（矢印改善済みだが根本解決ではない） |

### 発見3: 「最初の一歩」を踏み出せない層が想像以上に大きい

推定 **4,000人（40%）が LP を読んでも登録に至らない。**

| 踏み出せない理由 | 推定人数 | 対応可能な施策 |
|----------------|---------|-------------|
| 「自分には無理」という思い込み | 2,000人 | ⚠️ 同年代の成功体験談がもっと必要 |
| まだ検討段階（今すぐではない） | 1,500人 | △ リターゲティング基盤なし |
| 操作方法がわからない | 500人 | ⚠️ ビデオチュートリアルが効果的 |

### 発見4: セグメント間の「対立する要求」

| 要求 | S5の要望 | S3の要望 | 現在の対応 |
|------|---------|---------|-----------|
| アニメーション | もっとリッチに | 不要・邪魔 | ✅ prefers-reduced-motion で解決 |

| デザイン | ミニマル・余白重視 | 大きな文字・コントラスト | ✅ レスポンシブで両立 |

## 6. セグメント別のページ体験フロー分析

### S2（安心感最重視層・3,000人）のユーザージャーニー

```
LP到着 → Trust Badge確認（✅安心）→ 共感セクション（✅共感）
→ 比較表で理解（✅納得）→ 安心セクション（✅安全と判断）
→ FAQ確認（✅退会・料金に納得）→ ユーザーの声（✅同世代の声で安心）
→ Waitlist登録（✅「暗号化」の表示で最後の不安解消）
```
**ボトルネック:** 運営情報がないため、推定800人がここで離脱

### S3（IT苦手層・3,000人）のユーザージャーニー

```
LP到着 → フォント大きい（✅読める）→ Sticky CTAが常時見える（✅迷わない）
→ 共感セクション（✅「自分のこと！」）→ 「話すだけ」に安心（✅簡単そう）
→ Waitlist → メール入力+年代選択（✅シンプル）→ 登録完了
→ Chat → クイックリプライで返信（✅入力不要！）
```
**ボトルネック:** Signupの都道府県セレクト（推定1,200人が苦労）

### S4（自信欠如層・1,000人）のユーザージャーニー

```
LP到着 → 「写真なしでもOK」に安堵（✅参加できる）
→ 共感セクション（✅涙が出るほど共感）→ 「小さな一歩」に勇気づけられる
→ しかし…登録ボタンまでスクロールする間に不安が戻る
→ FAQ確認 → 「退会いつでも」に少し安心
→ 登録…するかどうかで数日悩む
```
**ボトルネック:** 即時行動に至らない。リターゲティングやメルマガ導線が必要
